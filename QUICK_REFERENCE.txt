## 🔧 Manual Startup (3 terminals)

### Terminal 1: Backend
```bash
cd backend && npm start
# Output: Server Running on PORT 5000
```

### Terminal 2: Frontend
```bash
cd frontend && npm run dev
# Output: Local: http://localhost:5173
```

### Terminal 3: Admin
```bash
cd admin && npm run dev
# Output: Local: http://localhost:5174
```

## ⚙️ Environment Configuration

### Backend Port
File: `backend/.env`
```env
PORT=5000
```

### Frontend API URL
File: `frontend/.env`
```env
VITE_API_URL=http://localhost:5000/api
```

### Admin API URL
File: `admin/.env`
```env
VITE_API_URL=http://localhost:5000/api
```

## 🐛 Quick Troubleshooting

| Issue | Solution |
|-------|----------|
| Port already in use | Change PORT in .env and vite.config.js |
| Cannot connect to API | Check backend is running on port 5000 |
| MongoDB connection failed | Check internet and MONGO_URI in .env |
| "Cannot find module" | Run `npm install` in the folder |

## 🚀 Production Deployment

### Step 1: Deploy Backend (Railway/Render)
```
1. Go to railway.app or render.com
2. Connect GitHub
3. Set environment variables
4. Deploy
```

### Step 2: Update Frontend API URL
Edit `frontend/.env`:
```env
VITE_API_URL=https://your-backend.railway.app/api
```

### Step 3: Deploy Frontend (Vercel)
```
1. Go to vercel.com
2. Connect GitHub
3. Deploy
```

### Step 4: Deploy Admin (Vercel)
Edit `admin/.env`:
```env
VITE_API_URL=https://your-backend.railway.app/api
```
Deploy to Vercel
